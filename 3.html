* JS
- 사용자와 상호작용을 하는 언어
- HTML을 제어하는 언어
- 웹페이지를 동적으로 만들어줌
- HTML은 1+1을 그대로 '1+1'로 표시하지만 JS는 2로 표현함 즉, 유동적임
- color.js 처럼 코드화 시켜서 유지보수를 용이하게 가능
- script src='color.js' 처럼 명시할 것.

* document
- document.querySelector('abc') : 태그 이름이 abc인 태그를 선택
- document.querySelector('#abc') : 태그 id가 abc인 태그를 선택
- document.querySelector('#abc').value : 태그 id가 abc인 태그의 value값을 가져옴
- .write : 화면에 표시
- document.querySelectorAll('a') : 태그 이름이 a인 태그 모두 선택

* 이벤트
- onclick : 클릭했을 때 "..."안의 JS코드를 실행
- onchange : 변화가 있을 때 "..."안의 JS코드를 실행
- onkeydown : 키가 눌러졌을 때 "..."안의 JS코드를 실행
- onmouseover : 마우스를 범위 안에 올렸을 때 "..."안의 JS 코드를 실행
- onmouseout : 마우스가 범위 밖에 나갔을 때 "..."안의 JS 코드를 실행
- onload : 브라우저 로딩이 끝났을 때 "..."안의 JS 코드를 실행

* 명령어
- 개발자모드에서 console창에서 JS 가능 (현재 웹페이지가 대상)
- "", '' 모두 쓰임
- 'abc'.length : 문자의 길이
- 'abc'.toUpperCase() : 대문자로 변경
- 'abc'.indexOf('a') : 문자 abc에서 a의 위치
- 'abc'.trim() : 문자열의 공백 제거
- "abc" + "de" : 문자열 합치기

* 데이터 타입
- boolean
- null
- undefined
- number
- string
- symbol

* 변수
- var x=1; var y=1; 처럼 대입연산자를 이용해서 변수 넣을 수 있음

* 리팩토링
- 코드를 좀 더 효율적으로 변경하여 유지보수를 용이하게 만듬
- 자기 자신을 가리키는 코드는 this. 로 간단히 표현할 수 있음
- 'var target = document.querySelector('body');'
- alist[i] 처럼 배열로 선언 가능
- alist[i].style.color = 'blue' 처럼 변경 가능

* 매개변수, 인자
- function sum(a, b) {...} sum(2,3); 라고 있을때,
- a,b는 매개변수(parameter) 2,3은 인자(argument)라고 한다.

* 객체
- 배열 생성엔 [] 대괄호
- 객체 생성엔 {} 중괄호
- var coworkers = {
"programmer": "egoing",
"designer": "leezche" };
- 이렇게 선언을 함
- "data scientist" 라는 값을 넣고 싶지만 공백이 있어서 넣지 못할 때,
- coworkers["data scientist"] = "taeho"; 이렇게 하면 가능
- 객체의 값을 순차적으로 가져올 때는 for(var key in cowrkers){...}를 씀
- coworkers.showAll() 처럼 객체에 소속된 함수는 method
- 객체 내의 programmer, designer 같은 것들은 Property라고 함

* 라이브러리, 프레임워크
- 다른 사람과 협력하는 모델
- 라이브러리 : 도서관의 뜻, 부품을 가져오는 느낌
- 프레임워크 : 만들려는 것의 공통적인 것들이 있고, 달라지는 부분만 수정해서 만드는 반제품 늬앙스.
- jQuery : JS에서 가장 유명, 생산성이 높아짐, CDN(Content Delivery Network)을 지원하기 때문에 script src에만 달아주면 서버비 없이 무료로 사용 가능 / JS의 생산성을 향상시켜주는 JS 라이브러리(자주 사용되는 로직들의 재활용, 유통 가능하도록 만든 로직들의 묶음)

* UI, API
- UI(User Interface) : 사용자가 목적달성을 위해 마주하는 조작체계
- API(Application Programming Interface) : alert 같이 프로그래머의 개발을 돕기위해 마련된 도구 및 환경

* TIP
- document : 웹페이지에 있는 어떤 태그를 삭제하고 싶거나 어떤 태그에 자식 태그를 추가하고 싶 때 (찾을 수 없다면 DOM)
- window : 웹브라우저 자체를 제어(현재 웹페이지의 주소, 새 창열기, 브라우저 크기 알기 등)
- ajax : 웹페이지를 리로드하지 않고 변경하고 싶다면
- cookie : 웹페이지가 리로드 되어도 현재 상태를 유지하고 싶을 때
- offline web application : 인터넷에 끊겨도 동작하는 웹페이지 제작
- webRTC : 화상 통신 웹 앱을 만들 때
- speech API : 사용자의 음성을 인식하고 음성으로 정보를 전달
- webGL : 3차원 그래픽으로 게임과 같은 것을 만들 때
- webVR : 가상현실
