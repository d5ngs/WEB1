WEB과 민영이

* 인터넷이 동작하기에는 최소 2대의 컴퓨터가 있어야 한다.


* Web Server가 설치된 컴퓨터에는

  1. info.cern.ch라는 주소를 부여
  2. index.html이라는 파일을 지정
  3. 서버 라고 할 수 있고, 클라이언트에게 response함
  4. index.html을 찾고 내용을 읽어서 클라이언트에게 신호를 보냄


* Web Browser가 설치된 컴퓨터에는

  1. http://info.cern.ch/index.html 이라는 주소를 입력할 수 있음
  2. 클라이언트 라고 할 수 있고, 서버에게 request함
  3. 서버로부터 신호가 왔을 때, 웹브라우저는 신호를 읽고 웹페이즈를 화면에 출력


* 우리는 늘 웹브라우저와 익숙하게 지내고 있다.
* 이번에는 웹서버로 눈을 돌려보자. 익숙해지는 것이다.
* 본인의 PC에 웹서버 설치 VS 웹서버를 제공해주는 웹호스팅


* Web Hosting

  1. git의 계정을 만들고 로그인
  2. Repository 생성(initialize this repository with a README도 생성)
  3. 업로드
  4. settings - GitHub Pages 탭에서 master branch - Save
  5. 내 웹페이지를 나타낼 주소가 나옴! 나는 https://d5ngs.github.io/WEB1/ 임


* Web Server 설치

  1. 아파치, 비트나미 설치
  2. 비트나미 매니저를 이용해 웹서버를 키고 끌 수 있음
  3. http://localhost = http://localhost/index.html,http://127.0.0.1/index.html 'localhost'라는 도메인 이름은 '내 컴퓨터'라는 의미
  4. 여기서 index.html은 htdocs 디렉토리내에 있음
  5. 웹서버를 쓰는 http://127.0.0.1/index.html VS 웹서버를 안쓰는file:///D:/job/web/index.html 는 사용자 입장에서는 큰 차이가 없음
  6. 하지만 웹서버를 쓰는 http://는 HyperTextTransfer Protocol의 약자로
    웹페이지를 전송하기 위해서 만든 통신규약임
    사실 file://처럼 자신의 컴퓨터에 있는 파일을 웹서버를 통해서 가져오는 것은 실제로 의미가 없음
  7. 그렇다면 127.0.0.1 VS 192.168.0.13
    127은 자기 자신을 의미하는 대명사
    후자는 하나의 컴퓨터를 가리키는 고유명사
    후자의 IP주소를 다른 PC에서 접속한다면 접속이 될 것이다! (단, 같은 네트워크를 사용!)
